# RuLetka

## Цель
Создать современную чат-рулетку, которая соединяет пользователей не случайно, а на основе их интересов и предпочтений, используя искусственный интеллект для анализа профилей собеседников.

## Описание решения
#### Создание пользователя
    - Регистрация нового пользователя (имя, пол, дата рождения, страна).
    - Заполнение профиля (выбор хобби, уникальное описание).
    - Панель поиска собеседника на главной странице.

#### Выдача (подходящие собеседники)
    - Система анализирует профиль пользователя.
    - Искусственный интеллект подбирает наиболее подходящего собеседника по параметрам профиля и предпочтениям.
    - При совпадении параметров двух пользователей, они соединяются в видеочат.

#### Созвон
    - Пользователь нажимает "Начать созвон".
    - При совпадении параметров происходит соединение в отдельной комнате.
    - После завершения звонка можно перейти к следующему собеседнику или завершить сессию.

## Схема высокоуровневой архитектуры

├── backend - бэкенд  
│   ├── api - REST  
│   │   ├── alembic - миграции для бд клиента (Postgres)  
│   │   └── app - сервисы FastAPI  
│   │       ├── auth - авторизация  
│   │       ├── expirience - информация пользователя  
│   │       └── rooms - основной функционал чат рулетки  
│   ├── common - общие для бэкенда скрипты  
│   │   ├── core - конфиги  
│   │   ├── schemas - pydantic схемы  
│   │   └── storage - всё для работы с различными бд  
│   │       ├── click_migrations - миграции для ClickHouse  
│   │       ├── models - модели данных  
│   ├── data_processor - сервис с ИИ моделью, генерирующей векторное описание пользователя  
│   │   ├── handlers  
│   │   └── models - ИИ модели  
│   │       └── rubert-tiny2  
│   └── pg_consumer - consumer для обращений к postgres-у  
│       └── handlers  
├── frontend - фронтенд  
  

## Компромиссы
    - Необходимость хранения и обработки персональных данных (вопросы приватности).
    - Возможные задержки в подборе собеседников при низкой онлайн-активности.
    - Баланс между скоростью соединения и качеством совпадений по интересам.

## Альтернатива - Классическая случайная чат-рулетка без учета интересов.
    - Ручной выбор собеседника из списка (менее анонимно, но больше контроля).
    - Использование только простых фильтров (пол, возраст, страна) без AI-анализа.

## Вопросы
    - Как обеспечить безопасность и приватность данных пользователей?
    - Какие дополнительные параметры могут улучшить качество подбора?
    - Как мотивировать пользователей заполнять профиль максимально подробно?

## Оценка
    - Ожидается повышение удовлетворенности пользователей за счет более релевантных собеседников.
    - Потенциал для роста аудитории за счет уникального подхода к подбору.
    - Возможность интеграции дополнительных функций (например, тематические комнаты, мини-игры).
